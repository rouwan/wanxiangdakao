## 一、mpvue 项目启动与编译

### 1.1 Node 版本：必须用 Node 10

**现象**：Node 14/24 均报 `Cannot read property 'loaded' of null` 或 `No such module: http_parser`

**根因**：mpvue → Webpack 3 → loader-runner 3 不兼容 Node 14+ 的内部 API

**解决**：使用 Node 10.24.1 便携版（`D:\Java\node10\`）

```bash
D:\Java\node10\node.exe build/build.js    # 构建
D:\Java\node10\node.exe build/dev-server.js  # 开发模式（监听文件变化）
```

### 1.2 node-sass：mpvue 必须用 node-sass，不能用 Dart Sass

**现象**：`Module build failed: Cannot find module 'node-sass'`

**根因**：mpvue 内部的 `sass-loader@7.0.3` 写死了 `require("node-sass")`，sass-loader 7 不认识 Dart Sass

**解决**：创建 fake `node-sass` 包，底层桥接到 Dart Sass

```
v3/前台前端/node_modules/node-sass/index.js  →  require('sass') 的包装
```

⚠️ **不要删掉这个假的 node-sass**，否则下次 npm install 后编译必炸

### 1.3 SCSS 中 `rpx` 单位不能出现在颜色函数里

**现象**：`Module build failed: $alpha: Expected 0.42rpx to have no units`

**报错位置**：`src/scss/hybird.scss:67` — `rgba(#7b70ba, 0.42rpx)`

**修法**：`rpx` 去掉 → `rgba(#7b70ba, 0.42)`

### 1.4 微信开发者工具基础库版本

**现象**：`Cannot read property 'call' of undefined` at `webpackJsonp`

**根因**：微信基础库 3.16.1 与 mpvue/Webpack 3 的分包加载机制不兼容

**持久化修法**：在 `project.config.json` 里固定：
```json
"libVersion": "3.5.8"
```
**导入时要用**父目录 `v3/前台前端/`（project.config.json 的 `miniprogramRoot: "dist/"` 会自动指向 dist）

### 1.5 DevTools 缓存问题

**现象**：改了源码但 DevTools 显示的还是旧版本

**解决三步**：
1. 关掉 DevTools 里的项目
2. 删除 `dist/` 目录
3. 重编 `node build/build.js`
4. 重新导入项目

⚠️ **不要只用编译按钮**——当文件被锁时，编译器的覆盖写入可能失败

## 二、后端启动与数据库

### 2.1 服务启动顺序

```
MySQL(3306) → Redis(6379) → 游戏后端JAR(8008) → 管理后端JAR(8005)
```

后端启动后等 4 秒才可用。

MySQL 启动命令：`D:\Java\mysql\bin\mysqld.exe --console`
Redis 启动命令：`D:\Java\redis\redis-server.exe --port 6379`

### 2.2 数据库缺表/缺列排查方法

**现象**：接口返回 500 `No message available`

**排查流程**：
1. 用 curl 调接口 + 带 Cookie（用 dev_test_01 登录拿 SessionId 后传 `Cookie: JSESSIONID=xxx`）
2. 查 `_out.txt`（后端 stdout 日志）里的 `Exception` 或 `BadSqlGrammar` 或 `Unknown column`
3. 根据错误信息建表/加列

**本项目中已补建的缺失项**：

### 2.3 MySQL sql_mode 问题

**现象**：`In aggregated query without GROUP BY, expression #2... incompatible with sql_mode=only_full_group_by`

**修法**：
```sql
SET GLOBAL sql_mode='STRICT_TRANS_TABLES,NO_ZERO_IN_DATE,NO_ZERO_DATE,ERROR_FOR_DIVISION_BY_ZERO,NO_ENGINE_SUBSTITUTION';
```

### 2.4 Java 进程残留

**现象**：重编 JAR 后端口仍被占用、mvn clean 删不掉 target 里的 JAR

**处理**：用 Python 杀进程（Bash 的 taskkill 有编码问题）
```python
subprocess.run(['taskkill', '/F', '/IM', 'java.exe'], capture_output=True)
sleep(3)  # 必须等
```
验证端口释放：
```python
s = socket.socket(); s.settimeout(1)
s.connect(('localhost', 8008))  # 连得上说明没释放
```

## 四、匹配引擎问题排查

### 4.1 机器人不匹配

**现象**：WebSocket 连接成功，心跳正常，但永远卡在"匹配中"

**检查清单**：
1. 后端日志 `_out.txt` 有没有 `[MATCH]` 日志行？（我已加了）
2. 机器人段位在不在用户匹配范围内？
   ```sql
   SELECT paragraph FROM wxdk_robotparagraph;
   -- 如果全是1但用户是段位3，匹配范围[2-4]就没有机器人满足
   ```
3. MySQL sql_mode 对不对？（`only_full_group_by` 会导致匹配成功但创建游戏时崩溃）

### 4.2 前端 WebSocket URL 问题

**现象**：后端日志大量 `NumberFormatException: For input string: "un"`（即 code=undefined）

**根因**：flyio 配置了 `parseJson: false`，`res.data` 是字符串，`res.data.Result` 取不到值，WebSocket URL 里 code 变成 `undefined`

**修法**：Dev 模式下先 `JSON.parse(res.data)` 再取字段

## 五、Bash 编码问题

**现象**：Bash 执行含中文路径的命令全部 exit 127

**铁律**：涉及中文路径的操作**必须走 Python 脚本**，不用 Bash。

**模板**：
```python
import subprocess, os
project = os.path.join('d:/workspace/workspace_2/wanxiangdakao', '万象大考整理', ...)
r = subprocess.run([...], cwd=project)
```
