# 数据库表关联关系说明

> 来源于 48 个 DAO Java 文件的 JOIN/SELECT 分析。注意：所有关联均为**应用层软引用**，数据库层面无外键约束。

---

## 核心关联枢纽（以 userId 为轴心）

本项目以 `wxdk_user.id`（用户ID）为绝对核心，几乎所有 wxdk_user* 表都通过 `userId` 字段指向它：

```
wxdk_user (id)
 ├── wxdk_userlastlogin (userId)
 ├── wxdk_userparagraph (userId) ────────── wxdk_paragraph (level=paragraph)
 │    └── wxdk_userparagraphhistory (userId)
 ├── wxdk_userability (userId) ──────────── wxdk_uplevel (grade+1=level)
 ├── wxdk_usercombat (userId)
 ├── wxdk_usercombatday (userId)
 ├── wxdk_usercombatdetail (userId)
 ├── wxdk_userdaysign (userId) ──────────── wxdk_daysign ────── wxdk_goods
 ├── wxdk_userdaytask (userId) ──────────── wxdk_daytask
 │    └── wxdk_userdaytaskrecord (userId)
 ├── wxdk_userdaychallenge (userId)
 ├── wxdk_userweekchallenge (userId) ────── wxdk_weektitle (titleId)
 ├── wxdk_userbook (userId) ─────────────── wxdk_book / wxdk_uplevel
 ├── wxdk_userlucknumber (userId) ───────── wxdk_lucknumber ─── wxdk_goods
 ├── wxdk_userprize (userId) ───────────── wxdk_prizeratio ─── wxdk_goods
 ├── wxdk_useremail (userId) ────────────── wxdk_email
 ├── wxdk_userfriend (userId, withUserId) ─ wxdk_user / wxdk_userparagraph
 ├── wxdk_userequipment (userId)
 ├── wxdk_userticket (userId)
 ├── wxdk_userformid (userId)

 ├── wxdk_usertoxcx (userId)

 ├── wxdk_usergamestatisrank (userId) ───── wxdk_user / wxdk_robot (endWinUserId)
 ├── wxdk_usergamestatisroom (userId)

 │    └── wxdk_userhitcatrecord (userId)
 ├── wxdk_userspecialsubject (userId) ───── wxdk_specialsubject ── wxdk_subject
```

## 对局域关联

```
wxdk_game ──────────────── 1v1 对局（detail JSON 存完整对局）
wxdk_gamerank (userId, withUserId) ──────── 排位对局记录
wxdk_gameroom (userId, roomId) ──────────── 房间混战记录
wxdk_robotgamerank (userId, withUserId) ── 机器人对局记录
```

这些表之间没有显式 JOIN（都通过 JSON 或用户ID间接关联），业务逻辑在 Service 层组合。

## 机器人域（负 ID 约定）

机器人的 `wxdk_robot.id` 在关联表中以**负值**形式出现，与真实用户的正值 `wxdk_user.id` 共用 `userId` 字段：

```
wxdk_robot (id=5)
 ├── wxdk_robotparagraph (userId=-5)
 ├── wxdk_robotgamestatisrank (userId=-5)
 ├── wxdk_robotdayrecord (userId=-5)
 ├── wxdk_robotgamerank (userId=-5)
 ├── wxdk_robotaction
 ├── wxdk_robotbasicinfo
 └── wxdk_robotchange
```

## 重要技术发现

| 发现 | 说明 |
|------|------|
| **0 个数据库外键** | 107 张表全部无 DDL 级外键约束，全靠 Java 应用层 JOIN |
| **负 ID 机器人** | 机器人 ID 取负值存入 userId 字段，区分真人和 Bot |
| **旧式逗号 JOIN** | 部分 DAO 使用 `FROM a, b` 而非 `JOIN`，见于 SubjectDao 等 |
| **wxdk_paragraph 双 JOIN** | 排位查询中常 JOIN 两次：算当前段位 + 晋级/降级段位 |
| **wxdk_subjectratio 子查询** | 正确率通过子查询 `(SELECT rightNum/totalNum ...)` 实时计算 |

## 快速理解三条主线

搞懂这三条链路，你就懂了这个项目的 80%：

### 链路 1：用户登录 → 排位匹配
```
wxdk_user → wxdk_userlastlogin → wxdk_userparagraph → wxdk_paragraph
```
- 主要看 DAO: [UserDao](万象大考整理/wxdk备份/.../dao/user/UserDao.java), [UserParagraphDao](万象大考整理/wxdk备份/.../dao/user/UserParagraphDao.java)

### 链路 2：答题对战 → 结算
```
wxdk_subject → wxdk_subjectoption → wxdk_gamerank
                     ↓
           wxdk_subjectratio (更新正确率)
                     ↓
           wxdk_userparagraph (更新段位/星级)
```
- 主要看 DAO: [SubjectDao](万象大考整理/wxdk备份/.../dao/user/SubjectDao.java), [GameRankDao](万象大考整理/wxdk备份/.../dao/user/GameRankDao.java)

### 链路 3：每日活跃 → 能力成长
```
wxdk_userdaysign → wxdk_userdaytask → wxdk_userability → wxdk_uplevel
```
- 主要看 DAO: [UserDaySignDao](万象大考整理/wxdk备份/.../dao/user/UserDaySignDao.java), [UserAbilityDao](万象大考整理/wxdk备份/.../dao/user/UserAbilityDao.java)
