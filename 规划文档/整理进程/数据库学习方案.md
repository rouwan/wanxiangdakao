# 附：新手看表方法论

> 适用：后端小白，第一次接触项目数据库。
> 核心思路：**一张表分三步看 —— 先在 DBeaver 看结构，再看数据长什么样，最后对照 Java 代码看业务逻辑。**

---

## 三步看懂一张表（以 wxdk_user 为例）

### 第一步：DBeaver 看表结构

在 DBeaver 左侧找到表 → 双击，右侧会打开 5 个标签页：

| 标签页 | 看的重点 |
|--------|----------|
| **Properties** | 表的基本信息：引擎（InnoDB）、字符集（utf8mb4）、行数 |
| **Data** | 实际数据，一眼看出这个表存了什么 |
| **Columns** | 列名 → 类型 → 是否允许 NULL → 默认值 → 注释 |
| **Constraints** | 主键、外键、唯一约束 |
| **DDL** | 建表的完整 SQL 语句（CREATE TABLE ...） |

**新手顺序**：先看 Columns（搞清楚有哪些字段），再看 Data（看一眼实际数据），最后扫一眼 DDL（看一下完整的建表语句，包括索引）。

### 第二步：看 Data 页 —— 理解"存了什么"

wxdk_user 的 Data 页你会看到类似：

| id | openId | nickname | sexDesc | city | headImgUrl | isTest |
|----|--------|----------|---------|------|------------|--------|
| 1 | oABC123... | 小明 | 男 | 广州 | https://... | 0 |

直观看出：**这张表存的是一行一行的用户**，每个用户有微信 openId、昵称、性别、城市、头像。

### 第三步：对照 DAO 源码 —— 理解"怎么用"

打开 [UserDao.java](万象大考整理/wxdk备份/user/vientianetestappendapi/src/main/java/com/example/vientianetest/dao/user/UserDao.java)，逐行看 SQL：

**这条 SQL：**
```sql
SELECT id, openId, nickname, sexDesc, city, province, country, headImgUrl, isTest
FROM wxdk_user
WHERE openId = :openId
```
→ 说明通过 **微信 openId 查用户**，返回完整的用户资��字段。

**这条 SQL：**
```sql
INSERT INTO wxdk_user(openId, createTime) VALUES(?, now());
```
→ 说明**新用户注册时只插 openId**，其余字段后续更新。

**这条 SQL：**
```sql
UPDATE wxdk_user SET nickname = ?, sexDesc = ?, city = ?, province = ?, country = ?, headImgUrl = ?, modifyTime = now() WHERE id = ?
```
→ 说明用户**每次授权登录时会更新 6 个资料字段**。

**这条 SQL：**
```sql
SELECT COUNT(1) FROM wxdk_user WHERE id = ? AND (nickname != ? OR headImgUrl != ? OR sexDesc != ? ...)
```
→ 说明系统会**比对资料是否变化**来判断用户是否更新了微信资料。

### 三步总结

| 步骤 | 操作 | 回答的问题 |
|------|------|-----------|
| Columns | DBeaver 点列标签 | 有哪些字段？什么类型？ |
| Data | DBeaver 点数据标签 | 实际数据长什么样？ |
| DAO 源码 | 看对应的 `*Dao.java` | 谁在读写这个表？怎么读写的？ |

### 阶段一：核心对战流程（19 张表） ⭐ 优先级最高

> **目标**：理解"用户登录 → 匹配 → 答题对战 → 结算 → 排位升降"这一条完整链路

| 模块 | 表数 | 表名 |
|------|------|------|
| 用户身份 | 2 | `wxdk_user`, `wxdk_userlastlogin` |
| 排位赛 | 4 | `wxdk_userparagraph`, `wxdk_paragraph`, `wxdk_rankseasontime`, `wxdk_userparagraphhistory` |
| 对局记录 | 3 | `wxdk_game`, `wxdk_gamerank`, `wxdk_gameroom` |
| 对战统计 | 3 | `wxdk_usergamestatisrank`, `wxdk_usergamestatisroom`, `wxdk_robotgamestatisrank` |
| 题库核心 | 6 | `wxdk_subject`, `wxdk_subjectoption`, `wxdk_subjecttype`, `wxdk_subjectratio`, `wxdk_subjectweight`, `wxdk_wordlibrary` |
| 系统配置 | 1 | `wxdk_dataconfig` |

**阅读顺序建议**：
1. 先看 `wxdk_user` → 用户长什么样
2. 再看 `wxdk_subject` + `wxdk_subjectoption` → 题目长什么样
3. 然后 `wxdk_game` / `wxdk_gamerank` → 一场对局怎么记录
4. 最后 `wxdk_userparagraph` + `wxdk_paragraph` → 排位怎么升降

**看完能回答的问题**：
- 用户怎么注册/登录？
- 一场对局从匹配到结束的数据流是怎样的？
- 段位星级的计算逻辑是什么？
- 题目的难度是怎么判定的？

## 执行建议

1. **用 DBeaver 的 ER 图功能**：右键表 → View Diagram，能看到外键关系和表间关联
2. **对照 DAO 源码看**：每张表在 `万象大考整理/wxdk备份/user/vientianetestappendapi/src/main/java/com/example/vientianetest/dao/` 下对应一个 DAO 文件，SQL 语句直接写了业务逻辑
3. **按阶段递进**：看完阶段一再看阶段二，避免跳跃
4. **文档记录**：看的过程中如果有新发现，补充到 `规划文档/整理进程/数据库说明.md`

## 验证方式

- 阶段一看完后：能对着 `wxdk_game.detail` 的 JSON 还原一场对局的完整过程
- 阶段二看完后：能从 `wxdk_userability` 推算一个用户当前的等级和体力
- 阶段三看完后：能解释机器人如何根据 `simpleRatio/commonRatio/hardRatio` 控制答题正确率
