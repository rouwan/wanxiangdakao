# 题库题型分析

> 来源：`wxdk_subject` 表 | 数据库：MySQL 5.7 `wxdkgamedo`

---

## 一、题目存储结构

所有题型共用一套表结构，靠 `modeType` 字段区分：

```
wxdk_subject (题目主表)
  └── wxdk_subjectoption (选项表，通过 subjectId = id 关联)

出题 SQL：SELECT * FROM wxdk_subject WHERE modeType = X
选项 SQL：SELECT * FROM wxdk_subjectoption WHERE subjectId = Y
```

另有成语分表 `wxdk_subject_chengyu` 和诗词分表 `wxdk_subject_sici`，但与主表 1:1 冗余，实际出题不走它们。

## 六、modeType 详细定义（来自 Subject.java 注释）

```java
// 1:图加文, 2:歌曲加文, 3:纯文字, 4:判断, 5:成语,
// 6:单词, 7:五言标题上, 8:五言标题下, 9:七言标题上, 10:七言标题下
```

### 每种 modeType 的交互逻辑

| modeType | 前端展示 | 选项数 | 特殊字段 |
|----------|---------|--------|---------|
| 1 | 上方大图 + 下方选项 | 4 | `image` |
| 2 | 音频播放 + 选项 | 4 | `audio` |
| 3 | 纯文字 | 4 | 无 |
| 4 | 命题 + 对/错 | 2 | 无 |
| 5 | 成语解释 + 选字区 | 4 但选字 | `author`/`siciTitle` 有分表 |
| 6 | 单词翻译 | 4 | 无 |
| 7-10 | 诗词上下句 | 4 | `author`/`siciTitle` 有分表 |

## 七、V4 简化建议

V4 阶段只需支持 2 种模式：

| 优先级 | modeType | 理由 |
|--------|----------|------|
| P0 | 3 纯文字 | 占 50%，出题成本最低 |
| P1 | 1 图加文 | 占 7%，图文配合更有趣 |

判断题、成语、单词、诗词可以后续再加。先拿纯文字跑通对战。
