## 一、前台前端（微信小程序 `user/dakao`）

- **框架**：mpvue 1.0（Vue 2.x → 微信小程序编译）
- **状态管理**：Vuex（3 个模块：root + friendBattle + socketStore）
- **HTTP 客户端**：flyio（类 Axios，支持小程序）
- **生产 API 地址**：`https://ggbsq.xingpanwang.com/vientianetest/mobile/`
- **WebSocket**：两个端点——`/wsRank`（排位）、`/wsRoom`（房间）

### 2.3 核心游戏引擎

#### 2.3.1 排位匹配引擎（WebSocketRankService）

```
匹配队列收集 5 秒
    ↓
按段位范围匹配（±1-2 段位差）
    ↓
排除最近对手（UserRankMatchRecordRedis 记录）
    ↓
排除 3 局内已匹配过的玩家
    ↓
无人可匹配 → 补机器人
第 5 局 → 补机器人
连输 3 局 → 补机器人
    ↓
创建 Game 对象，通知双方
```

#### 2.3.2 计分规则

```
基础得分 = 100
速度加分 = (剩余秒数 / 10) × 10
最后一题暴击：基础分 × 30
宝典加成 = 题目学科对应的宝典等级加成%
总得分 = 基础分 + 速度分 + 宝典加成
```

#### 2.3.3 机器人行为模拟

| 参数 | 简单 | 普通 | 困难 |
|------|------|------|------|
| 正确率 | `simpleRatio` | `commonRatio` | `hardRatio` |
| 答题延迟下限 | `simpleTime.min` | `commonTime.min` | `hardTime.min` |
| 答题延迟上限 | `simpleTime.max` | `commonTime.max` | `hardTime.max` |
| 错误选项 | 随机选一个错误选项 | 随机 | 随机 |

- 机器人用户 ID 为**负数**（如 -1, -2...），以此与真实用户区分
- 每种难度有独立的正确率和答题时间范围

#### 2.3.4 结算逻辑

```
比赛结束后：
1. 判定胜负（得分高者胜，平分则平局）
2. 更新段位（胜+星，败-星，王者段位用 MMR）
3. 更新战斗力（各学科分别累计）
4. 更新连胜/连败计数
5. 更新游戏统计（总局数、胜率、最高单局分）
6. 更新金币（胜+15，败+5）
7. 更新体力（消耗 1 点）
8. 更新 Redis 排行榜 ZSet
9. 晋级赛判定（黄金/钻石/王者段位需要晋级赛）
```

### 3.2 核心机制

| 机制 | 说明 |
|------|------|
| 动态菜单 | 登录后从 `/sys/menu/nav` 拉取菜单树，根据用户角色过滤权限，动态注入 Vue Router |
| 权限控制 | 每个菜单项关联 `perms` 权限字符串（如 `sys:user:list`），按钮级权限通过 `v-hasPerm` 指令控制 |

| 验证码 | `/captcha.jpg?uuid=xxx` 生成 4 位字符验证码，登录需填写 |

## 五、数据访问模式对比

### 游戏后端（vientianetestappendapi）

```
Controller → Service → DAO（NamedParameterJdbcTemplate）
                         ↓
                    手写 SQL + BasicResultObject（JSON String）
```

- 特点：无 ORM，全部手写 JDBC
- DAO 返回 `String`（手动拼 JSON）
- Service 直接操作 DAO，无事务注解

### 管理后端（renren-fast）

```
sys 模块: Controller → Service → MyBatis-Plus BaseMapper
small 模块: Controller → Service → DAO（NamedParameterJdbcTemplate）
                                   ↓
                              手写 SQL + StringBuilder 拼 JSON
```

- sys 模块用了 MyBatis-Plus（标准 RBAC CRUD）
- small 模块全部用原生 JDBC（手写 SQL + `@Component` DAO）
- 软删除：`update table set flag = 3 where id = :id`
- 少数表物理删除：`delete from table where id = :id`
